## Release notes guidance

When preparing a release, prefer short entries grouped under:

- `Added`
- `Changed`
- `Fixed`
- `Security`
- `Docs`

Keep each item user-facing. If a change only affects internal refactoring, mention it only if contributors or self-hosters would care.
